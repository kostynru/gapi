-------------------------------------------------------------------------------

Getting API - Realise Candidate 
1. Добавлена поддержка ответа в формате json.
2. Добавлена поддержка обработки ответа от метода в виде одномерного массива.
3. Код был значительно оптимизирован.
4. Изменена структура базы данных. 
5. Теперь все ошибки возвращаются в виде XML, независимо от выбранного формата. Описание ошибки находится в контейнере
<error>.
6. Добавлены новые, пока неизвестные баги.

-------------------------------------------------------------------------------

Getting API - Realise Version
1. Добавлена полная поддержка ответа в формате json.
2. Добавлена поддержка ответа в виде многомерных массивов, но только в формате json.
3. Теперь все массивы возвращаются корректно как в json, так и в XML, в том числе и многомерные (См. руководство, 
"Общая информация об использовании API, пункт 6). И все же мы рекомендуем возвращать массивы в виде json.
4. Теперь не обязательно использовать ООП для создания методов, вы можете использовать простое объявление функций
(напомним метод = функция), но мы настоятельно рекомендуем использовать ООП.
5. Теперь вы можете осуществлять запросы к API в виде www.example.com/api/пространство.метод&параметры...
Обратите внимание на то, что после указания пространства и метода стоит знак '&', а не '?'.
6. В дистрибутив включен пример сайта с регистрацией пользователей, приложений и выводом списка приложений текущего
залогинившегося пользователя. Сайт находится в папке app_reg.
7. Испытания Alfa версии закончены.
8. Все выводы ошибок были объеденены в единную функцию error_to_xml(). Есть возможность вывода ошибки в виде json, но
стандартным способом остается xml.
9. Внимание! Начиная со следующей версии будет введен режим, требующий декларирования всех методов сервера. Это является
мерой безопастности. Способ декларирования будет указан в мануале для следующей версии.
10. Теперь, если при попытке авторизации ключ признается истекшим, то из базы удаляются все ключи созданные данным 
приложением.

Getting API - Realise Version (Исправления от 18.12.2012)
1. Улучшена безопасность. Добавлена невозможность SQL-инъекций.
2. Теперь параметры передаются стандартным способом при помощи '?'. Отмена изменения №5 в предыдущей версии. Для запроса
исползуйте следующий вид - www.example.com/пространство.метод?параметры...
3. Исправлена куча багов.
4. Исправлена система вывода ошибок.
